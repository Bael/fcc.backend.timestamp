API Basejump: Timestamp microservice

This microservice implements next user stories:

  • I can pass a string as a parameter, and it will check to see whether that string contains either a unix timestamp or a natural language date (example: January 1, 2019)
  • If it does, it returns both the Unix timestamp and the natural language form of that date.
  • If it does not contain a date or Unix timestamp, it returns null for those properties.

Example usage

  • https://tranquil-coast-90658.herokuapp.com/2019-01-01
  • https://tranquil-coast-90658.herokuapp.com/1546300800
  • https://tranquil-coast-90658.herokuapp.com/January%201,%202019

Output

  • { "unix":1546300800, "natural":"January 1, 2019" }
